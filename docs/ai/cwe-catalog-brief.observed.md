# CWE Catalog Brief — observed subset

Filtered from `Microsoft.CodeAnalysis.Sarif.Taxonomies.CweTaxonomy.brief.md`
(MITRE CWE 4.20; 969 entries) to the **384 CWE identifiers that at
least 1 of 6 surveyed production SAST scanners tags in its published
rule metadata** (CodeQL, Semgrep, Bandit, gosec, Snyk Code, Veracode
SAST).

**Purpose.** This is the broader empirical "what scanners actually
tag" reference — the escalation set when a finding's CWE doesn't fit
any entry in the companion `cwe-catalog-brief.shared.md` (the
55-entry default for classification contexts). The full MITRE catalog
(969 entries) is the deeper fallback when even this broader set lacks
a match.

**Selection threshold.** Union of CWE identifiers across all six
surveyed tools (≥1 tag). This is the empirical floor: every CWE the
security-tooling industry has visibly invested in detection logic for,
based on the tools we inventoried. Compare to the shared brief's
≥4-of-6 threshold, which selects the high-confidence converged
subset. See `cwe-tool-coverage-methodology.md` for the full tier
distribution and rationale.

**This is a snapshot, not a steady state.** Tool rule catalogs evolve,
MITRE adds and deprecates entries, and our surveyed-tool set will
grow. Refresh against the methodology's recipe periodically.

**Companion files in this directory.**
- `cwe-catalog-brief.shared.md` — narrower 55-CWE "converged"
  default (≥4 of 6 tools). Use this by default for AI classification;
  consult `observed.md` only when no shared-brief entry fits.
- `per-tool-coverage.json` — machine-readable per-CWE tool-set
  provenance for the full 396-CWE union.
- `cwe-tool-coverage-methodology.md` — how this was made, what was
  found, how to use it.

**MITRE status retained as-is.** Many entries the industry actively
detects are marked `Incomplete` by MITRE (SSRF, MissingAuthz,
Observable Timing Discrepancy among them). MITRE maturity status is
documentation maturity, not detection maturity; we do not filter on
it.
